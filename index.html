Nodestore – A MySQL- and Node.js-based Inventory Management System

Synopsis
            node customer
            node manager
            node supervisor

The following commands are supported:

  • node customer:  List items for sale and purchase items.
  • node manager:  List items for sale, view items with low inventory, add to inventory, and create new items in the system.
  • node supervisor:  View product sales by department and create new departments.
Installation

Nodestore requires that you have MySQL installed on your computer (download MySQL).  You will also need to have MySQL Workbench installed in order to import an existing database (download MySQL Workbench).  Once both MySQL and MySQL Workbench are installed and running, to install and set up Nodestore:

  1. In a Terminal (or equivalent) window, clone the Nodestore GitHub repository with:
    git clone https://github.com/powerpcg5/nodestore.git
  2. cd nodestore;
  3. vi .env, and append the following lines to the file:
    # MySQL database password
    MYSQL_PASSWORD=yourMySQLpassword
  4. npm init -y;
  5. Install required NPM modules:  npm i dotenv mysql inquirer (Note:  As of the time of this writing (6/21/2019) the current version of the inquirer NPM package, 6.4.0, causes the console to hang due to nested inquirer.prompt calls; if you encounter this problem, install a previous working version instead, e. g., npm i inquirer@6.3.1  Fixed in inquirer version 6.4.1 of 6/22/2019);
  6. Start MySQL Workbench, and connect to the database (e. g., Database → Connect to Database…);
  7. Copy the initial schema file schema.sql in the nodestore directory (you can also view the source here), and then paste and run the commands in MySQL Workbench;
  8. Import the products.csv file in the nodestore directory (you can also view the source here) into the nodestoreDB (e. g., by clicking on the import icon next to Export/Import:, selecting products.csv, and following the prompts, with Use existing table: set to nodestoredb.products); and
  9. Finally, import the departments.csv file in the nodestore directory (you can also view the source here) into the nodestoreDB (e. g., by clicking on the import icon next to Export/Import:, selecting departments.csv, and following the prompts, with Use existing table: set to nodestoredb.departments).

You should now be able to use run node customer, node manager, or node supervisor.  If you get an Error: ER_ACCESS_DENIED_ERROR: Access denied for user 'root'@'localhost' (using password: NO) error, check that you have specified the correct password in .env, and that .env is in the nodestore directory.

Notes

The most involved MySQL query required for this application was that used to view the aggregate sales by department, which involved joining the product database and the department database on the department key; the overall query looked like the following:

USE nodestoreDB;
SELECT departments.dept_id, departments.department,
  SUM(products.product_sales) AS total_sales, departments.overhead,
  SUM(products.product_sales) - departments.overhead AS net_total_sales
FROM products
INNER JOIN departments
ON products.department = departments.department
GROUP BY departments.dept_id, departments.department

The name Nodestore was inspired by the ‘Apple Store’ and is a nod to its being powered by Node.js.  The ‘-store’ part of the name is a reference to the data store (storage), which is handled by MySQL.
